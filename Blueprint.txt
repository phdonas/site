Sistema de prioridades
Cada funcionalidade neste blueprint tem uma prioridade explícita:

P1 — CRÍTICO
Implementar ANTES da primeira linha de código. São regras que previnem bugs graves documentados em projetos similares.

P2 — ALTA
Implementar nos 60 dias. Fazem parte do MVP e do roadmap principal.

P3 — MÉDIA
Implementar junto com o módulo correspondente se o custo for baixo; caso contrário, logo após o lançamento.

P4 — PÓS-LANÇAMENTO
Implementar após validação com usuários reais. Requerem dados históricos ou massa crítica de alunos.

P5 — LONGO PRAZO
Roadmap de médio prazo. Alta complexidade ou dependência de infraestrutura que ainda não existe.

## ══ REGRAS ABSOLUTAS ══

### Segurança — P1 CRÍTICO (implementar antes de qualquer código)
- Rate limiting APENAS em /api/* — NUNCA em rotas de páginas, assets, CSS ou JS
  (aplicar em todas as rotas derruba o site: 1 visita dispara 15-20 requisições de assets)
- /admin/login deve ficar FORA do Route Group (protected) para evitar loop infinito
  Estrutura correta: /app/(auth)/login | /app/(protected)/admin/...
- Slugs são SEMPRE ASCII lowercase sem acentos e sem caracteres especiais
  CORRETO: &apos;gestao-comercial&apos; | ERRADO: &apos;gestão-comercial&apos; (quebra 55+ links internos)
- Validar senha em TODO endpoint de autenticação — nunca verificar apenas o e-mail
- NUNCA expor ANTHROPIC_API_KEY no frontend — sempre via /api/
- Verificar vencimento do plano em TODA rota protegida de conteúdo
- Assinatura da plataforma (planos_acesso tipo=plataforma) tem prioridade sobre matrícula

### Identidade visual e código
- NUNCA hardcodar cores — sempre lib/design-tokens.ts
- NUNCA hardcodar pilares — sempre buscar da tabela pilares
- Mobile-first: componente funciona em 375px antes de desktop
- Simuladores HTML ficam no HostGator — link protegido com token PHP
- PDFs exclusivos: Supabase Storage com signed URLs de 60min

### Glossário — USAR SEMPRE ESTES TERMOS
- Módulo = agrupamento de aulas. NUNCA &apos;seção&apos;, &apos;unidade&apos; ou &apos;capítulo&apos;
- Ferramenta = tool SaaS com output direto (formulário→texto). Salva em ferramenta_resultados
- Simulador = roleplay conversacional multi-turno. Salva em simulacoes
- Biblioteca = tabelas modulos/aulas com curso_id NULL + tabelas cursos_modulos/modulos_aulas

6. Proteção dos Simuladores HTML — Token PHP
🔒  Sem esta proteção, qualquer pessoa que souber a URL direta acessa o simulador sem estar logada.

Como funciona
1. Aluno clica &apos;Abrir Simulador&apos; → plataforma gera JWT de 15 min. 2. URL enviada: phdonassolo.com/simuladores/arquivo.html?token=xyz. 3. token-check.php valida antes de exibir. 4. Sem token válido → redirect para login.

6.1 Crie token-check.php no HostGator
&lt;?php
// /public_html/simuladores/token-check.php
$secret = &apos;SEU_SECRET_AQUI&apos;; // mesmo valor de SIMULADOR_TOKEN_SECRET no Vercel
$token = $_GET[&apos;token&apos;] ?? &apos;&apos;;
if (empty($token)) {
  header(&apos;Location: https://alunos.phdonassolo.com/login&apos;); exit;
}
$parts = explode(&apos;.&apos;, $token);
if (count($parts) !== 3) {
  header(&apos;Location: https://alunos.phdonassolo.com/login&apos;); exit;
}
$payload = json_decode(base64_decode(strtr($parts[1], &apos;-_&apos;, &apos;+/&apos;)), true);
$sig = hash_hmac(&apos;sha256&apos;, $parts[0].&apos;.&apos;.$parts[1], $secret, true);
$expected = rtrim(strtr(base64_encode($sig), &apos;+/&apos;, &apos;-_&apos;), &apos;=&apos;);
if ($parts[2] !== $expected || $payload[&apos;exp&apos;] &lt; time()) {
  header(&apos;Location: https://alunos.phdonassolo.com/login&apos;); exit;
}
// Token válido — continua exibindo o simulador
?&gt;

6.2 Adicione no início de cada simulador HTML
&lt;?php require_once &apos;token-check.php&apos;; ?&gt;

🤖  PROMPT — CRIAR O BANCO (Semana 3, primeiro prompt ao Claude Code)
&quot;Crie o schema completo do banco de dados no Supabase. Execute o SQL abaixo em sequência — são 44 tabelas. Após criar todas as tabelas, habilite Row Level Security e crie as políticas da seção 7.3. Confirme listando as tabelas criadas.&quot;

7.2 SQL Completo — 44 Tabelas
-- ════════════════════════════════════════
-- GRUPO 1: PERFIL E PILARES
-- ════════════════════════════════════════
CREATE TABLE profiles (
  id UUID REFERENCES auth.users(id) PRIMARY KEY,
  email TEXT NOT NULL, nome TEXT NOT NULL, foto_url TEXT,
  papel TEXT DEFAULT &apos;aluno&apos; CHECK (papel IN (&apos;aluno&apos;,&apos;admin&apos;)),
  bio TEXT, linkedin TEXT,
  lgpd_aceito_em TIMESTAMPTZ, lgpd_versao TEXT,
  cadastro TIMESTAMPTZ DEFAULT NOW(), ultimo_acesso TIMESTAMPTZ
);
CREATE TABLE pilares (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  nome TEXT NOT NULL UNIQUE, slug TEXT NOT NULL UNIQUE,
  descricao TEXT, ativo BOOLEAN DEFAULT TRUE, ordem INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE TABLE profiles_pilares (
  profile_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
  pilar_id UUID REFERENCES pilares(id) ON DELETE CASCADE,
  PRIMARY KEY (profile_id, pilar_id)
);

-- ════════════════════════════════════════
-- GRUPO 2: CURSOS
-- ════════════════════════════════════════
CREATE TABLE cursos (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  titulo TEXT NOT NULL, slug TEXT UNIQUE NOT NULL,
  descricao TEXT, thumbnail TEXT,
  video_apresentacao_id TEXT,
  o_que_voce_vai_aprender JSONB,
  para_quem_e_este_curso JSONB,
  requisitos JSONB,
  nivel TEXT DEFAULT &apos;todos&apos; CHECK (nivel IN (&apos;iniciante&apos;,&apos;intermediario&apos;,&apos;avancado&apos;,&apos;todos&apos;)),
  idioma TEXT DEFAULT &apos;pt-BR&apos;,
  carga_h INTEGER DEFAULT 0,
  atualizado_em TIMESTAMPTZ,
  preco NUMERIC(10,2) DEFAULT 0,
  preco_promo NUMERIC(10,2),
  preco_promo_ate TIMESTAMPTZ,
  proximo_curso_id UUID REFERENCES cursos(id),
  status TEXT DEFAULT &apos;rascunho&apos; CHECK (status IN (&apos;rascunho&apos;,&apos;publicado&apos;,&apos;arquivado&apos;)),
  codigo_editorial TEXT,  -- ex: &apos;PHD_015&apos; — ponte com planilha editorial
  created_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE TABLE cursos_pilares (
  curso_id UUID REFERENCES cursos(id) ON DELETE CASCADE,
  pilar_id UUID REFERENCES pilares(id) ON DELETE CASCADE,
  PRIMARY KEY (curso_id, pilar_id)
);

-- ════════════════════════════════════════
-- GRUPO 3: MÓDULOS (com suporte a biblioteca)
-- ════════════════════════════════════════
-- curso_id NULL = módulo na biblioteca (reutilizável entre cursos)
-- Módulo vinculado diretamente a 1 curso: usar curso_id normal
-- Módulo reutilizado em N cursos: curso_id NULL + tabela cursos_modulos
-- GLOSSÁRIO: Módulo = agrupamento de aulas. NUNCA &apos;seção&apos; ou &apos;capítulo&apos;.
CREATE TABLE modulos (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  curso_id UUID REFERENCES cursos(id) ON DELETE CASCADE,  -- NULL = biblioteca
  titulo TEXT NOT NULL,
  descricao TEXT,
  conteudo_html TEXT,
  ordem INTEGER NOT NULL DEFAULT 0,
  status TEXT DEFAULT &apos;publicado&apos; CHECK (status IN (&apos;rascunho&apos;,&apos;publicado&apos;)),
  codigo_editorial TEXT  -- ponte com planilha editorial
);

-- Vínculo módulo da biblioteca ↔ cursos (com ordem independente por curso)
-- Permite que &apos;Técnicas de Negociação&apos; apareça no curso A na posição 3
-- e no curso B na posição 5, sem duplicar o conteúdo
CREATE TABLE cursos_modulos (
  curso_id UUID REFERENCES cursos(id) ON DELETE CASCADE,
  modulo_id UUID REFERENCES modulos(id) ON DELETE CASCADE,
  ordem INTEGER NOT NULL,
  PRIMARY KEY (curso_id, modulo_id)
);

-- ════════════════════════════════════════
-- GRUPO 4: AULAS (com suporte a biblioteca)
-- ════════════════════════════════════════
-- tipo define o componente renderizado pelo player:
-- &apos;video&apos;    → youtube_id (YouTube IFrame API)
-- &apos;texto&apos;    → conteudo_html (rich text)
-- &apos;quiz&apos;     → tabela quizzes
-- &apos;download&apos; → aulas_materiais
-- &apos;link&apos;     → link_externo (nova aba)
-- &apos;live&apos;     → link_externo + data_live
CREATE TABLE aulas (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  modulo_id UUID REFERENCES modulos(id) ON DELETE CASCADE,  -- NULL = biblioteca
  titulo TEXT NOT NULL,
  tipo TEXT DEFAULT &apos;video&apos; CHECK (tipo IN (&apos;video&apos;,&apos;texto&apos;,&apos;quiz&apos;,&apos;download&apos;,&apos;link&apos;,&apos;live&apos;)),
  ordem INTEGER NOT NULL DEFAULT 0,
  youtube_id TEXT,
  conteudo_html TEXT,
  link_externo TEXT,
  data_live TIMESTAMPTZ,
  duracao_min INTEGER DEFAULT 0,
  pergunta_reflexao TEXT,
  transcricao TEXT,
  transcricao_json JSONB,  -- [{start_s, text}] para sincronização
  gratuita BOOLEAN DEFAULT FALSE,
  status TEXT DEFAULT &apos;publicado&apos; CHECK (status IN (&apos;rascunho&apos;,&apos;publicado&apos;)),
  codigo_editorial TEXT  -- ponte com planilha editorial (ex: &apos;PHD_015&apos;)
);

-- Vínculo aula da biblioteca ↔ módulos (com ordem independente por módulo)
CREATE TABLE modulos_aulas (
  modulo_id UUID REFERENCES modulos(id) ON DELETE CASCADE,
  aula_id UUID REFERENCES aulas(id) ON DELETE CASCADE,
  ordem INTEGER NOT NULL,
  PRIMARY KEY (modulo_id, aula_id)
);

-- ════════════════════════════════════════
-- GRUPO 5: MATERIAIS
-- ════════════════════════════════════════
CREATE TABLE materiais (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  titulo TEXT NOT NULL, descricao TEXT,
  tipo TEXT CHECK (tipo IN (&apos;pdf&apos;,&apos;xlsx&apos;,&apos;pptx&apos;,&apos;docx&apos;,&apos;simulador&apos;,&apos;link&apos;,&apos;zip&apos;)),
  tipo_acesso TEXT DEFAULT &apos;curso&apos; CHECK (
    tipo_acesso IN (&apos;todos&apos;,&apos;pilar&apos;,&apos;curso&apos;,&apos;combinado&apos;)),
  storage_path TEXT,
  url_externa TEXT,
  gratuito BOOLEAN DEFAULT FALSE,
  codigo_editorial TEXT,  -- ponte com planilha editorial
  created_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE TABLE materiais_pilares (
  material_id UUID REFERENCES materiais(id) ON DELETE CASCADE,
  pilar_id UUID REFERENCES pilares(id) ON DELETE CASCADE,
  PRIMARY KEY (material_id, pilar_id)
);
CREATE TABLE materiais_cursos (
  material_id UUID REFERENCES materiais(id) ON DELETE CASCADE,
  curso_id UUID REFERENCES cursos(id) ON DELETE CASCADE,
  PRIMARY KEY (material_id, curso_id)
);
CREATE TABLE aulas_materiais (
  aula_id UUID REFERENCES aulas(id) ON DELETE CASCADE,
  material_id UUID REFERENCES materiais(id) ON DELETE CASCADE,
  ordem INTEGER DEFAULT 0,
  PRIMARY KEY (aula_id, material_id)
);

-- ════════════════════════════════════════
-- GRUPO 6: MATRÍCULAS E ACESSO
-- ════════════════════════════════════════
CREATE TABLE matriculas (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id UUID REFERENCES profiles(id),

-- ════════════════════════════════════════
-- FUNÇÃO: VERIFICAÇÃO DE ACESSO
-- ════════════════════════════════════════
CREATE OR REPLACE FUNCTION tem_acesso_curso(p_user_id UUID, p_curso_id UUID)
RETURNS BOOLEAN AS $$
BEGIN
  IF EXISTS (SELECT 1 FROM planos_acesso
    WHERE user_id=p_user_id AND tipo=&apos;plataforma&apos; AND ativo=TRUE
    AND (vencimento IS NULL OR vencimento&gt;NOW())) THEN RETURN TRUE; END IF;
  IF EXISTS (SELECT 1 FROM planos_acesso
    WHERE user_id=p_user_id AND tipo=&apos;curso&apos; AND curso_id=p_curso_id AND ativo=TRUE
    AND (vencimento IS NULL OR vencimento&gt;NOW())) THEN RETURN TRUE; END IF;
  IF EXISTS (SELECT 1 FROM matriculas
    WHERE user_id=p_user_id AND curso_id=p_curso_id AND status=&apos;ativo&apos;)
    THEN RETURN TRUE; END IF;
  RETURN FALSE;
END; $$ LANGUAGE plpgsql SECURITY DEFINER;

Regra
Onde aplicar
O que acontece se ignorar
Rate limiting APENAS em /api/*
middleware.ts

/admin/login FORA do Route Group protegido
/app/(auth)/login — NUNCA dentro de (protected)
Layout verifica sessão → redireciona para login → herda layout → loop infinito. Painel admin inacessível.
Slugs sempre ASCII sem acentos

Validar senha em todo endpoint de auth

ANTHROPIC_API_KEY apenas no servidor
